BASICS OF HTML

*HTML stands for HyperText Markup Language.
*HyperText is the link of web and Markup is the layout information of the web or documents.


HTML5

The DOCTYPE declaration is simple "!DOCTYPE html". "audio","video" tags are supported for playing audio and video files. The HTML5 supports the API for identifying the geographic location. The tags that are removed from HTML5 are "applet,big,center,font,frame,strike". It allows JavaScript to run in background. The "canvas"tag is for 2D drawing. The need for external plugin is reduced. The HTML is a "plain text file", where the file is reffered as rich text due to the tags.

TAGS

Strings enclosed within angular brackets are known as Tags.The slash (/) indicates the end of that tag.
For example "</html>" is reffered for end of that tag.


HEAD AND BODY

The HTML has two parts such as HEAD and BODY.
The head part consist of the title of the web page. The contents of the head part is not visible in client area. The body part is used to create the look and feel of web page. It's contents are visible in client area. The body part has elements


ELEMENTS

There are several body elements for text formatting such as "p, br, prev, div, a"


TEXT FORMATTING

P TAG is used to define a new paragraph. If we give space between two lines in a single paragraph, the space will not be shown in that paragraph. If you want to give that line break , we can create two sperate paragraph.
BR TAG is used to give single line break. Since it is a self closing tag, it is an "Empty element". If we want more three blank lines to be inserted after some text then we can put "br" tag three times.
PRE TAG is used to override a browser's normal white spacing processing. It is a preformatted tag. Most browsers will not perform word wrapping on this text even of a line is too long, to fit within. Instead, it will provide horizontal scroll bar. Monospace font will be used.
DIV TAG is used to make division of sections in the XHTML documents.

TEXT TAGS

Three types of font style can be used here.
  • B TAG is used to display text in bold.
  • STRONG TAG is also used to display the text in bold.
  • I TAGis used to display in italics.
  • EM TAG is used to display as emphasised text.
  • DFN TAG is used for definiton term.
  • SMALL TAG makes the text appear smaller in browser.
  • U TAG underlines the text.
  • STRIKE TAG is used to striked through.
  • DEL TAG encloses deleted text.
  • INS TAG encloses inserted text.
  • SUB TAG displays a text as subscript.
  • SUP TAG displays a text as superscript.

FONT SETTING

We can set the font, size and color of the text in webpage. The "BASEFONT TAG" is used for this purpose. The syntax is <basefont face=font_name>. This tag is not supported in HTML5 , XHTML. It was valid till HTML4. If you want to change the font, size and color, you should go with CSS. To display a text colorful on the webpage, we use the "FONT FACE" tag. We can also set the color and size of the font using its attributes in double quotes.
This tag is not supported in HTML5.


BACKGROUND COLOR SETTING

The background color can be set using the attribute "BGCOLOR" in the body tag. The syntax will be <body bgcolor="red" >. Here the color can be specified using the color name or hex code. HTML supports several colors like "Aqua, Black, Blue, Fuchsia, Gray, Green, Lime, Maroon, Navy, Olive, Purple, Red, Silver, Teal, White, Yellow".


HORIZONTAL RULES

The "HR TAG" is used to display a horizontal rule in the webpage. This line appears below the preceeding HTML content and above the content following the hr element. It is an empty element.


UNRECOGNISED ELEMENTS AND ATTRIBUTES

*HTML has a unique feature in it. If we make any typographical mistake such as p, the browser will still attempt to display the entire webpage.
*For attributes with unrecognised names, the browser acts as if the attribute is not present at all.
*For unrecognised element names, the browser displays the content of the element as if the markup were not present.
*In the title tag, if we mistype as "<titl>", the contents of the title tag will be displayed in the client area.
*Eventhough the HTML didn't throw an err, if we convert the HTML document into a plaintext , it would likely produce an error.
To avoid this situation , it is a good idea to check the validity of the HTML. One simple way to perform validation checking is to use an "HTML VALIDATOR" such as the one available at the W3C [W3C-VAL]. It also conforms to the standards of the HTML version you are using.


SPECIAL CHARACTERS

To display the symbols, we use a type of markup known as a "reference". This reference always begins with ampersand sign and ends with semicolon sign. There are two types of reference.
*A reference which uses a mnemonic name for the character referenced, is called a "entity reference".
*The hash sign follows the ampersand beginning the reference and is followed by the Unicode Standard value of the character, is known as "character reference".
*The entity reference of the following symbols "<, >,&, ", ', ©, ñ, α, ∀ are lt, gt, amp, quot, apos, copy, ntilde, alpha, forall".
*Another type of entity reference that is used frequently is "nbsp", that is "non breaking space character". This entity reference is useful for defining the space between two strings and informing browser for not performing the word wrapper between the string.
For example ,
Displaying without nbsp will be "WEB DEV".
Displaying with nbsp will be "WEB    DEV".

HYPERLINKS

There is a common practice to specify the web link in the web page. The link acts as a pointer to some web page or some resource.
The 'a' or anchor element is used for creating clickable link. The href attribute of an anchor element specifies the URL of a document to be requested via the HTTP GET method if the link corresponding to the anchor is clicked by the user
HYPERLINKS can be used by <a>, and by specifying the URL for href. <a> is used for beginning while </a> is used to end the web link.
The value assigned to href specifies the target of the link.

STEP 1: beginning should be <a href="" . Inside the double quotes mention the URL of desired link.
STEP 2: write some text that should act as a hyperlink.
STEP 3: end the web link </a>.

A link has two ends called anchors and a direction.
The link starts at the "source anchor" and points to the "destination anchor" which may be any web resource.

*Paired anchor tag is used for hyperlink. It has four attributes. 1, href - specifies the URL of the page the link goes to
2, id - specifies the identity of an anchor.
3, name - works same as id attribute
4, target - specifies where to open tbe linked file
     By default the hyperlink page gets opened in the browsers window. If we want to get that link opened in another window we can mention _target property. We can mention the target element by including the id attribute which can be used to identity it in an href attribute.
While defining the header we can include the id attribute, <h2 id="book"></h2>. Hence we can use that id at href as #book instead of link.
Various targets can be:
*_self loads the page into the current window.
*_blank loads the page into a new seperate browser window.
*_parent loads the page into the frame that is superior to the frame the residing hyperlink.
*_top cancels all frames and loads in full browser window.

TABLE

To construct a table using HTML we use <table> tag. Under the table tag we use thead , tbody tags to display the heading and body of the table.
The <tr> tag is used to hold the values to be printed in one row.
The <th> tag will be inside the thead tag. This tag will contain the heading of the table. It will be in bold.
The <td> tag will be inside the tbody tag. This tag will contain data to be stored in the table.
The <border> tag will be defined inside the table tag, so that the table will be displayed with borders.

FORM

The form tag has action attribute which specifies the URL to send form data to. It also has method attribute which specifies the type of HTTP request. The <input> tag is used to accept data from the user.
Some of the inputs are:
  • TEXT
    It is used to store text data.
    Syntax: type="text"
  • PASSWORD
    It is used to enter a secure password.
    Syntax: type="password"
  • PLACEHOLDER
    It is used for temporary text in input fields. It is generally accompanied by "text" and "password" attributes.
    Syntax: placeholder="insert-text-here".
  • BUTTON
    It is used to include button in the form.
    Syntax: type="button" value="insert-value-here"
  • SUBMIT BUTTON
    All the data will be submitted when it is clicked
    Syntax: type="submit"
  • CHECKBOX
    It is used to provide the ability to check multiple options.
    Syntax: type="checkbox"
  • RADIO BUTTON
    It allows one to choose a single option.
    Syntax: type="radio"
The select tag is used for every possible option to select, it uses an <option> tag.
The text areas tag is used for multi-line plaintext content with "rows" and "cols" attributes.
The label tag is used to add captions for individual items in a form. The content of label tag will be displayed in form. It also attributes like "for" and "id".
We also have validations to ensure that users fill out forms in correct format. The required attribute makes that field mandatory.
The email tag ensures that field contains @ symbol.

ATTRIBUTES

We have some global attributes in HTML. They are title, style.

QUOTATIONS, ABBREVIATIONS AND CITATIONS

BLOCKQUOTE TAG is used to difine block quotation. Q TAG is used to define a short quotation. Browses normally insert quotation marks around the quotes. ABBR TAG defines an abbreviation or an acronym, like HTML, CSS. While hovering on these abbreviations the expansions will be shown if we use the abbr tag.
ADDRESS TAG defines the contact information for the author/owner of the article. The information can be a email address, URL, physical address, phone number etc.
CITE TAG defines the title of the creative work i.e, it shows the text in italics format.
